# 编辑映射功能测试计划

## 测试环境

- **Blender 版本**: 3.6+
- **插件**: XPS to PMX（已同步）
- **XPS 模型**: 任何具有手指骨的 XPS 模型（如女性人物模型）

## 测试场景 1：基本编辑功能

### 前置条件
1. 在 Blender 中加载一个 XPS 模型（具有手指骨）
2. 选中骨架
3. 打开 XPS to PMX Mapper 面板
4. 点击"🔄 Auto Map Bones"完成自动映射

### 测试步骤

1. **打开树形结构面板**
   - [ ] 点击"🌳 MMD 骨骼树形结构"面板标签
   - [ ] 验证面板显示 MMD 骨骼树
   - [ ] 验证统计信息显示 49 个 MMD 骨骼和 177 个 XPS 骨骼

2. **找到问题骨骼**
   - [ ] 打开"⚠️ 未映射骨骼检查"面板
   - [ ] 查看问题骨骼列表（如果有的话）
   - [ ] 记录手指骨的映射状态

3. **打开编辑对话框**
   - [ ] 在树形结构中搜索"左手首"
   - [ ] 点击该骨骼以在详情面板中选中
   - [ ] 查看详情面板中的"🔗 XPS 映射信息"
   - [ ] 记录当前的 XPS 映射
   - [ ] 点击"✎ 编辑映射"按钮

### 预期结果
- [ ] 编辑映射对话框应该弹出
- [ ] 对话框显示"📋 编辑映射信息"，包括：
  - XPS 骨骼名称（例如 lThumb1）
  - 当前 MMD 映射（例如 センター）
- [ ] 对话框显示"👇 选择新的 MMD 骨骼映射"和下拉列表
- [ ] 下拉列表中包含所有 49 个 MMD 骨骼

## 预期问题和解决方案

### 问题 1：下拉列表为空
**可能原因**: MMD 标准骨骼文件未加载
**解决方案**: 检查 `mmd_standard_skeleton.json` 是否存在并有效

### 问题 2：编辑对话框未弹出
**可能原因**:
- 未选择 MMD 骨骼
- 未运行自动映射

**解决方案**: 按照上述前置条件确保已完成必要步骤

### 问题 3：修改不生效
**可能原因**: 全局配置 (`_TREE_STATE['editing_xps_mapping']`) 未正确设置

**解决方案**: 查看 Blender 系统控制台输出，检查错误信息

### 问题 4：冲突检测失效
**可能原因**: 映射对象比较逻辑有误

**解决方案**: 添加调试输出，打印冲突检测的过程
