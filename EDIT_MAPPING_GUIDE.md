# 编辑骨骼映射功能使用指南

## 功能概述

新的编辑映射功能（Method B：完整列表）允许用户手动修改 XPS 骨骼到 MMD 骨骼的映射关系。当自动映射出现错误时（例如手指骨映射到错误的位置），用户可以直接进行修正。

## 功能位置

编辑映射功能集成在三个地方：

### 1. 骨骼树形结构面板（🌳 MMD 骨骼树形结构）
- 在树形结构中看到每个 MMD 骨骼和对应的 XPS 映射
- 可以点击骨骼树中的骨骼名称来在详情面板中显示该骨骼的信息

### 2. 骨骼详情面板（骨骼详情）
- 显示选中 MMD 骨骼的详细信息
- 包含 XPS 映射状态、层级关系、权重信息等
- **编辑按钮**：底部的"✎ 编辑映射"按钮可以打开编辑对话框

### 3. 未映射骨骼检查面板（⚠️ 未映射骨骼检查）
- 列出所有有问题的骨骼（未映射或映射质量低）
- 可从此面板快速找到需要修正的骨骼

## 使用流程

### 步骤 1：找到需要修改的骨骼

**方式 A：通过树形结构查找**
1. 打开"🌳 MMD 骨骼树形结构"面板
2. 展开树形结构（点击"展开全部"或点击 ▼ 符号）
3. 找到映射有误的 MMD 骨骼（例如手指骨 左手首 显示映射到 センター）
4. 点击该骨骼名称以在详情面板中选中它

**方式 B：通过问题列表查找**
1. 打开"⚠️ 未映射骨骼检查"面板
2. 查看列出的问题骨骼
3. 找到需要修正的 XPS 骨骼

### 步骤 2：打开编辑对话框

1. 在"骨骼详情"面板中确保已选中要编辑的 MMD 骨骼
2. 点击"✎ 编辑映射"按钮
3. 编辑映射对话框会弹出，显示：
   - **编辑映射信息**：显示当前的 XPS 骨骼名称和现有 MMD 映射
   - **选择新的 MMD 骨骼映射**：一个下拉列表，包含所有 49 个 MMD 骨骼

### 步骤 3：选择正确的 MMD 骨骼

1. 点击"选择 MMD 骨骼"下拉菜单
2. 浏览或搜索所需的 MMD 骨骼名称
3. 点击选择它
4. 对话框底部会显示"✓ 已选择："以及新的映射目标

### 步骤 4：确认修改

1. 检查新映射是否正确：`{XPS骨骼名} → {新的MMD骨骼名}`
2. 点击对话框的"✓ 确定"按钮以应用修改
3. 点击"✗ 取消"以放弃修改

## 实际例子：修改手指骨映射

### 问题场景
自动映射将手指骨（如 lThumb1-3）错误地映射到 センター（中心控制骨），导致骨骼层级错误。

### 修正步骤

1. **打开树形结构面板**，搜索"左手首"
2. **点击"左手首"**以在详情面板中选中
3. **点击"✎ 编辑映射"按钮**打开编辑对话框
4. 编辑对话框显示：
   ```
   XPS 骨骼: lThumb1
   当前 MMD: センター
   ```
5. **选择正确的 MMD 骨骼**：
   - 点击下拉菜单，搜索"左手"
   - 找到"左手"或"左手首"
   - 选择它
6. **确认修改**：对话框显示 `lThumb1 → 左手首`
7. **点击"✓ 确定"**应用修改

### 验证修改

修改后，检查树形结构和统计信息：
- ✅ 树形结构面板中，左手首的映射应该改为 `lThumb1`
- ✅ 未映射骨骼检查面板中，问题骨骼数应该减少
- ✅ 统计摘要中，映射覆盖率应该提高

## 技术细节

### 修改时发生的事情

1. **映射更新**：XPS 骨骼的 `mmd_name` 字段被更新为新的 MMD 骨骼名称
2. **冲突处理**：如果新的 MMD 骨骼已从其他 XPS 骨骼映射过来，旧的映射会被清除
3. **验证检查**：系统自动验证新映射的父级关系是否正确
4. **UI 刷新**：所有面板会自动刷新以显示新的映射

### 数据结构

```python
# 在全局配置中存储的映射数据
{
  'xps_name': 'lThumb1',          # XPS 骨骼名
  'mmd_name': '左手首',            # 映射到的 MMD 骨骼名
  'confidence': 0.45,              # 自动检测的置信度（可能较低）
  'parent_xps': 'lHand',           # XPS 中的父级
  'parent_mmd': '左腕',            # 映射后的 MMD 父级
  'parent_mmd_expected': '左腕',   # 标准 MMD 层级中的期望父级
  'parent_match': true,            # 父级关系是否匹配
  'is_deform': true,               # 是否是变形骨
  'is_unmapped': false,            # 是否已映射
  'vertex_group_count': 150,       # 有权重的顶点数
  'user_notes': '修正自动映射错误' # 用户备注
}
```

## 常见问题与故障排查

### Q: 编辑映射对话框没有出现？
**A:**
- 确保已选中一个 MMD 骨骼（在详情面板中应显示该骨骼的信息）
- 确保已运行过"🔄 Auto Map Bones"自动映射
- 检查日志中是否有错误消息

### Q: 编辑后没有看到变化？
**A:**
- 打开"🌳 MMD 骨骼树形结构"面板，它应该实时显示新的映射
- 如果仍未更新，尝试刷新面板（折叠和展开）
- 检查"⚠️ 未映射骨骼检查"面板中的问题列表是否已更新

### Q: 能否撤销编辑？
**A:**
- 可以再次打开编辑对话框，将映射改回原来的 MMD 骨骼
- 系统支持多次编辑，每次都会覆盖前一次的映射

### Q: 编辑后权重会自动调整吗？
**A:**
- **不会**。编辑映射只更改骨骼名称和层级关系
- 权重调整需要在后续的"权重修复"阶段单独进行
- 确保先完成所有映射修正，再进行权重处理

## 最佳实践

1. **优先处理高权重的骨骼**
   - 检查"⚠️ 未映射骨骼检查"面板中的骨骼，按顶点数排序
   - 优先修正权重较大的骨骼

2. **检查父级关系**
   - 编辑映射后，系统会自动检查父级关系
   - 如果出现"⚠️ 父级不匹配"警告，检查新的映射是否合理

3. **批量修正相似的骨骼**
   - 手指骨、肋骨等有相似命名的骨骼可能需要一起修正
   - 例如，所有 lThumb* 骨应该映射到 左手首 的子骨骼

4. **保存修改**
   - 修改后，配置会保存在全局状态中
   - 建议在完成所有编辑后，保存为自定义预设（待后续功能支持）
