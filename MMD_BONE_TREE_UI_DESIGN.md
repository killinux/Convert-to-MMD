# MMD 骨骼树形结构友好展示设计

## 概述

这是一个**专门的、美观的、易于理解的 MMD 骨骼树形结构展示面板**。目标是让用户一眼看清楚：
- MMD 骨骼的完整层级关系
- 每个骨骼的属性（类型、是否变形、用途）
- 对应的 XPS 映射
- 层级之间的父子关系

## 📊 详细信息

### 每个骨骼行的结构

```
[展开符] [图标] 【骨骼名】 [星标] [状态图标]
└─ [说明文字]
   [XPS 映射: XXX] ✓  [权重: NNN]  [类型: XXX]  [父级验证]
   ┌─ 子骨骼 (N 个) [展开/折叠]
   │  [子骨骼快速预览]
```

### 组件详解

#### 1. **展开/折叠符号**
```
▼  展开状态（点击折叠）
►  折叠状态（点击展开）
-  无子骨骼
```

#### 2. **骨骼类型图标**
```
🔴 control    - 控制骨（不用于变形）
🟢 spine      - 脊柱/躯干骨
🔵 arm/leg    - 四肢骨
🟡 d_bone     - D-骨（变形配对）
🟣 ik         - IK 链
🌕 twist      - 扭转骨
👁️  eye       - 眼睛骨
⚪ center     - 中心骨
```

#### 3. **骨骼名称**
```
【日文名称】 - 使用日文名称，用【】框起来，显示重要性
例: 【上半身】【首1】【足D.L】
```

#### 4. **星标（权重指示）**
```
⬤⬤⬤⬤⬤  - 权重很大（> 5%）
⬤⬤⬤⬤   - 权重较大（2-5%）
⬤⬤⬤    - 权重中等（0.5-2%）
⬤⬤     - 权重较小（< 0.5%）
⭕      - 无权重数据
```

#### 5. **状态图标**
```
✓  绿色复选 - 正确映射
⚠️ 黄色警告 - 父级不匹配、信息不完整
❌ 红色叉   - 未映射、无效映射
❓ 灰色问号 - 信息不确定
```

#### 6. **说明文字**
```
└─ [骨骼用途的简短说明，帮助理解]

示例：
└─ 中心骨，主要运动控制点
└─ 上半身主控制，支撑手臂和脖子
└─ 腰部 Cancel 骨，消除腰部不必要的运动
```

#### 7. **XPS 映射信息**
```
[XPS 映射: 骨骼名称] ✓/⚠️/❌

✓ 正确映射 - 绿色
⚠️ 警告 - 黄色（通常是父级不匹配）
❌ 未映射 - 红色
```

#### 8. **权重信息**
```
[权重: NNN 个顶点组] 或 [权重: N.N%]

显示该骨骼影响的顶点数或权重百分比
如果是 0，显示 [权重: 0] 或 [无权重]
```

#### 9. **骨骼类型**
```
[类型: 变形骨] 或 [类型: 控制骨]

显示该骨骼是否参与网格变形
```

#### 10. **父级验证**
```
[父级: 上半身 ✓] 或 [父级: 腰 ⚠️]

✓ 正确 - 父级映射与 MMD 标准一致
⚠️ 警告 - 父级不匹配或不存在
```

#### 11. **子骨骼快速预览**
```
┌─ 子骨骼 (N 个) [▼展开] [►折叠]
│  【子骨1】【子骨2】【子骨3】...

折叠时显示子骨骼的快速预览列表
点击可快速展开或折叠
```

## 📱 三种显示模式

### 模式 1️⃣：完整树（默认）

**适合**: 第一次查看、需要了解全貌
**显示**: 所有信息

```
【全ての親】
└─ 【センター】
   [XPS 映射: root_hips] ✓  [权重: 456]  [变形骨]
   └─ 【グルーブ】
      [XPS 映射: groove] ✓  [权重: 0]
```

### 模式 2️⃣：简洁树

**适合**: 快速浏览、关注结构
**显示**: 仅骨骼名称和映射状态

```
【全ての親】
├─ 【センター】 ✓
│  └─ 【グルーブ】 ✓
│     └─ 【腰】 ✓
└─ ...
```

**适合**: 详细比对、按属性排序
**显示**: 表格形式

```
┌─────────────┬──────────────┬────────┬─────────┬────────┐
```

## 🔍 搜索和过滤

### 搜索功能

```
搜索框: [____________________]

用户输入时的行为：
1. 实时过滤骨骼名称
2. 自动展开包含搜索词的路径
3. 高亮匹配的骨骼名称
4. 显示匹配数量 "搜索结果: 5/48"

示例搜索：
"腰" → 显示所有包含"腰"的骨骼
      ├─ 【腰】(主骨)
      ├─ 【腰キャンセル.L】
      └─ 【腰キャンセル.R】

"ＩＫ" → 显示所有 IK 骨骼
      ├─ 【左足ＩＫ】
      ├─ 【右足ＩＫ】
      ├─ 【左足IK親】
      └─ ...

"❌" → 显示所有未映射的骨骼
      ├─ 【上半身捩】
      ├─ 【グルーブ】
      └─ ...
```

### 过滤选项

```
☑ 仅控制骨  - 只显示控制骨（管理/操作骨）
☑ 仅变形骨  - 只显示会影响网格的骨骼
☑ 仅 D-骨   - 只显示变形配对骨（足D.L等）
☑ 仅 IK 链  - 只显示 IK 骨骼
☑ 仅未映射  - 只显示没有 XPS 映射的骨骼
☑ 仅警告    - 只显示有问题的骨骼（⚠️ 或 ❌）
```

## 🎬 交互动画和反馈

### 展开/折叠动画

```
当用户点击 ▼ 符号时：
1. 符号平滑旋转: ▼ → ►
2. 子骨骼列表平滑滑出或收起
3. 记住用户的展开状态（下次打开保持）
4. 动画时间: 200ms
```

### 悬停效果

```
鼠标悬停在骨骼名称上时：
1. 背景色淡化突出
2. 右侧显示操作按钮（编辑、详情、高亮）
3. 底部显示简短的骨骼描述

示例：
【上半身】 ← 悬停时
  └─ 上半身主控制骨，支撑手臂和脖子，关键变形骨
     [✎ 编辑] [📍 详情] [👁️ 在3D高亮]
```

### 搜索高亮

```
用户搜索 "腰" 时：
1. 所有包含 "腰" 的骨骼名称被黄色高亮
2. 搜索路径自动展开（从父到子）
3. 非匹配的骨骼淡化显示
4. 实时更新计数: "搜索结果: 3/48"
```

## 🎯 用户交互流程

### 流程 1：快速浏览骨骼结构

```
1. 用户打开面板
   ↓
2. 看到完整的树形结构（默认全展开）
   ↓
3. 向下滚动，理解层级关系
   ↓
4. 点击 [简洁树] 模式，看到简化版本
   ↓
5. 点击 [表格] 模式，按属性排序比较
```

### 流程 2：查找特定骨骼

```
1. 用户搜索 "腰"
   ↓
2. 树形结构自动过滤，只显示包含"腰"的骨骼
   ↓
3. 搜索路径自动展开
   ↓
4. 匹配的骨骼被高亮
   ↓
5. 用户点击其中一个，右侧显示详情
```

### 流程 3：查看未映射骨骼

```
1. 用户勾选 "☑ 仅未映射"
   ↓
2. 树形自动过滤，仅显示 ❌ 未映射的骨骼
   ↓
3. 用户看到：3 个未映射骨骼
   ├─ 【上半身捩】
   ├─ 【グルーブ】
   └─ 【??某个骨骼】
   ↓
4. 用户点击任何一个骨骼
   ↓
5. 右侧显示详情和建议映射
   ↓
6. 用户点击 [✎ 编辑]，快速修正
```

### 流程 4：验证父子关系

```
1. 用户看到【上半身2】的详情
   ↓
2. 显示：[父级: 上半身1 ✓] (正确)
   ↓
3. 用户继续往下，看到【某个骨骼】
   ↓
4. 显示：[父级: 上半身2 ⚠️] (不匹配！)
   ↓
5. 详情面板提示：
   "警告: 父级应该映射到 '上半身2'
    但实际映射到 '上半身3'
    是否修正？[是] [否]"
```

### 展开/折叠状态管理

```python
class XPSPMXBoneTreeState:
    expanded_bones: Set[str] = set()  # 记住用户展开的骨骼

    def toggle_expand(self, bone_name: str):
        if bone_name in self.expanded_bones:
            self.expanded_bones.remove(bone_name)
        else:
            self.expanded_bones.add(bone_name)
```
