# 编辑映射功能实现总结

**状态**: ✅ 完成
**日期**: 2026-03-29
**方案**: Method B - 完整 MMD 骨骼列表

## 功能概述

实现了一个完整的骨骼映射编辑功能，允许用户手动修改 XPS 骨骼到 MMD 骨骼的映射关系。当自动映射出现错误时（特别是手指骨等细节），用户可以直接进行修正。

## 已实现的组件

### 1. 核心操作符（XPSPMX_OT_edit_bone_mapping）

**文件**: `mmd_bone_tree_ui.py` (第 261-400 行)

**功能**:
- `invoke()`: 打开编辑对话框
- `draw()`: 绘制对话框 UI，显示 MMD 骨骼下拉列表
- `execute()`: 处理映射更改并更新全局配置

**特性**:
- 使用 EnumProperty 生成动态下拉列表（所有 49 个 MMD 骨骼）
- 自动验证新映射的父级关系
- 检测和处理映射冲突（同一个 MMD 骨骼被多个 XPS 骨骼映射）
- 实时 UI 更新反映所有更改

### 2. MMD 骨骼枚举生成函数

**函数**: `_get_mmd_bones_enum(scene, context)`

**功能**: 从 mmd_standard_skeleton.json 动态生成所有 49 个 MMD 骨骼的枚举列表

**特点**:
- 按字母顺序排序
- 包含完整的骨骼描述（日文名称）
- 错误处理：无法加载时返回错误提示

### 3. UI 集成

**集成点**:
1. **骨骼详情面板** (XPSPMX_PT_bone_detail_panel)
   - 第 768 行：添加了"✎ 编辑映射"按钮
   - 显示当前 XPS 映射信息

2. **全局状态管理**
   - `_TREE_STATE['editing_xps_mapping']`: 存储当前编辑中的映射对象
   - 确保在对话框打开和关闭时正确保存/清除状态

### 4. 数据结构更新

**BoneMapping 类** (来自 mapping/data_structures.py):
```python
{
    'xps_name': str,              # XPS 骨骼名（如 'lThumb1'）
    'mmd_name': str,              # 映射到的 MMD 骨骼名（如 '左手首'）
    'confidence': float,          # 自动检测置信度（0~1）
    'parent_xps': str,            # XPS 中的父级
    'parent_mmd': str,            # 映射后的 MMD 父级
    'parent_mmd_expected': str,   # MMD 标准中的期望父级
    'parent_match': bool,         # 父级是否匹配
    'is_deform': bool,            # 是否是变形骨
    'is_unmapped': bool,          # 是否已取消映射
    'vertex_group_count': int,    # 顶点权重数
    'user_notes': str,            # 用户备注
    'source_info': str            # 映射来源信息
}
```

## 工作流程

### 用户操作流程

```
1. 打开"🌳 MMD 骨骼树形结构"面板
   ↓
2. 在树中找到需要编辑的 MMD 骨骼（或在"⚠️ 未映射骨骼检查"中找到）
   ↓
3. 点击骨骼在"骨骼详情"面板中选中
   ↓
4. 点击"✎ 编辑映射"按钮
   ↓
5. 编辑对话框弹出，显示：
   - 编辑信息（当前 XPS→MMD 映射）
   - 下拉列表（所有 49 个 MMD 骨骼）
   ↓
6. 用户从下拉列表选择新的 MMD 骨骼
   ↓
7. 点击"✓ 确定"应用更改（或"✗ 取消"放弃）
   ↓
8. UI 自动刷新，显示新的映射关系
```

### 系统内部流程

```
当用户点击"✎ 编辑映射"：
1. invoke() 获取当前选中的 MMD 骨骼名称
2. 在全局配置中查找映射到该 MMD 骨骼的 XPS 骨骼
3. 存储该映射对象到 _TREE_STATE['editing_xps_mapping']
4. 打开 invoke_props_dialog() 显示对话框

当用户在对话框中选择新的 MMD 骨骼：
1. EnumProperty 的 items callback 实时刷新列表
2. 用户的选择被存储在操作符的 selected_mmd_bone 属性中
3. draw() 显示选择的确认信息

当用户点击"✓ 确定"：
1. execute() 被调用
2. 检查新选择是否有效
3. 检测冲突：如果新 MMD 骨骼已从其他 XPS 骨骼映射，清除旧映射
4. 更新 XPS 映射对象的 mmd_name 字段
5. 设置 is_unmapped = false
6. 验证新映射的父级关系
7. 清除 _TREE_STATE['editing_xps_mapping']
8. 触发 UI 刷新（所有视图实时更新）
9. 报告成功信息
```

## 核心特性

### 1. 冲突处理
当用户将 XPS 骨骼 A 映射到 MMD 骨骼 X（而 X 已从 XPS 骨骼 B 映射过来）时：
- 系统检测到冲突
- 自动取消 B 的映射（标记为 is_unmapped=true）
- 发出警告信息
- B 重新出现在"⚠️ 未映射骨骼检查"面板中

**代码位置**: `execute()` 第 370-380 行

### 2. 父级关系验证
修改映射后，系统自动检查新映射的父级关系：
- 获取新 MMD 骨骼的定义（从 mmd_standard_skeleton.json）
- 获取 XPS 父级在映射中对应的 MMD 名称
- 比较是否匹配 MMD 标准
- 设置 parent_match 标志和 parent_mmd_expected

**代码位置**: `execute()` 第 381-390 行

### 3. 动态枚举列表
下拉列表包含所有 49 个 MMD 骨骼，通过回调函数动态生成：
- 排序后显示（易于查找）
- 每个项目包含骨骼名称和描述
- 支持 Blender 的本地化（可翻译）

**代码位置**: `_get_mmd_bones_enum()` 函数

### 4. UI 实时同步
所有关联的面板实时显示最新的映射信息：
- 树形结构：显示更新的 XPS→MMD 映射
- 详情面板：显示编辑后的映射关系
- 未映射列表：自动更新问题骨骼列表
- 统计信息：实时更新覆盖率

**实现方式**: 使用 `context.screen.areas` 遍历所有视图区域，调用 `area.tag_redraw()` 触发刷新

## 使用场景示例

### 场景 1：修正手指骨映射

**问题**: 自动映射将 `lThumb1` 映射到 `センター`，应该映射到 `左手首`

**解决方案**:
1. 在树形结构中找到 左手首
2. 点击它，在详情面板中显示其当前映射（`lThumb1`）
3. 点击"✎ 编辑映射"
4. 从下拉列表选择 左手首（或其他正确的位置）
5. 点击"✓ 确定"

**结果**: `lThumb1` 现在正确映射到 `左手首`

### 场景 2：修正扭转骨映射

**问题**: 自动映射创建了一个无法识别的扭转骨，位置错误

**解决方案**:
1. 在"⚠️ 未映射骨骼检查"中找到该扭转骨
2. 打开编辑对话框
3. 选择最接近的 MMD 扭转骨（或决定删除它）
4. 如果保留，选择合适的父骨骼位置

### 场景 3：处理变体骨骼

**问题**: XPS 模型包含标准版本中没有的额外骨骼（如额外的肋骨）

**解决方案**:
1. 找到无法自动映射的额外骨骼
2. 根据位置和用途，手动将其映射到最接近的 MMD 骨骼
3. 或选择"标记为忽略"（将其标记为 is_unmapped=true）

编辑映射功能成功实现了用户最初的需求：
> "怎么能设计得，加个功能，每个骨骼的转换有对对应关系，且能人工检查细节是否是对的"
