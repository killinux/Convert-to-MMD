# XPS to PMX UI 设计文档

## 概览

整个映射系统分为 **5 个面板**，核心是以下两个：

```
┌─────────────────────────────────────────────────────────┐
│ AUTO DETECTION (读取专用)                                │
│ ➜ 自动检测完成，显示摘要和统计信息                      │
└─────────────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────────────┐
│ ① MMD 骨骼映射编辑器 (USER EDIT)        ⭐ 核心面板    │
│ ➜ 树形结构显示 MMD 骨骼                                  │
│ ➜ 每个 MMD 骨骼旁显示对应的 XPS 骨骼                    │
│ ➜ 可编辑、可折叠、可搜索                                │
└─────────────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────────────┐
│ ② 未映射骨骼仪表板 (STATUS MONITOR)    ⭐ 核心面板    │
│ ➜ 列出所有未映射的 XPS 骨骼                              │
│ ➜ 显示顶点数和权重百分比                                │
│ ➜ 点击后在视口高亮                                      │
└─────────────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────────────┐
│ WEIGHT REPAIR (建议面板)                                 │
│ ➜ 权重修复建议和策略                                    │
└─────────────────────────────────────────────────────────┘
           ↓
┌─────────────────────────────────────────────────────────┐
│ VALIDATION & PREVIEW (最终验证)                         │
│ ➜ 完整性检查                                            │
└─────────────────────────────────────────────────────────┘
```

## 📐 面板 1：MMD 骨骼映射编辑器

### 设计目标
- 显示完整的 MMD 骨骼层级树
- 每个骨骼显示对应的 XPS 映射
- 支持快速编辑、搜索、折叠
- 实时反馈验证结果

### 详细说明

#### A. 骨骼树形结构

**折叠/展开机制**：
```
▼ 展开状态
├─ 骨骼内容完全显示
└─ 子骨骼列表可见

► 折叠状态
└─ 显示 "子骨: N 个 (已折叠)" 提示
   点击后展开
```

**骨骼行信息**：
```
[展开符] [骨骼名] (骨骼类型 | 变形/不变形)

例如:
▼ 上半身 (变形骨 | 变形)
```

#### B. XPS 映射行

```
XPS: [映射名称_________________________] ✓/⚠️ 状态 ⚙️ 编辑

状态图标:
✓ 绿色复选 - 正确映射
⚠️ 黄色警告 - 父级不匹配、信息不完整
❌ 红色叉 - 映射不存在、无效引用

编辑按钮:
⚙️ 点击打开编辑对话框
```

#### C. 信息行（折叠显示）

```
信息: [来源] | [置信度] | [编辑状态]

来源：
  - preset: 来自预设
  - auto_detect: 自动检测
  - user_edit: 用户手动编辑

置信度：
  - 100% (绿色) - 完全匹配
  - 90-99% (绿色) - 高置信
  - 70-89% (黄色) - 中等置信
  - <70% (红色) - 低置信

编辑状态：
  - 未编辑
  - 已修正错误
  - 用户自定义
```

#### D. 验证指示

```
父级: [XPS 父级名] → [MMD 父级名] ✓/⚠️

示例：
✓ 父级: pelvis → 腰 (正确)
⚠️ 父级: abdomenLower → 上半身2 (错误, 应该是腰)

权重: NNN 个顶点组 (如果有权重数据)

子骨: N 个 (展开/折叠)
```

**搜索下拉框**：
- 实时搜索 XPS 骨骼名称
- 显示相似度百分比
- 绿色 ✓ 标记推荐项
- "❌ [无映射]" 选项用于移除映射

**级联更新**：
```
当用户更改父级骨骼的映射时：
1. 新的 parent_mmd 会自动计算
2. 如果有子骨骼，询问是否级联更新
   "上半身1 的父级会从 '上半身' 变成 '新骨骼'，是否更新？"
3. 更新所有受影响的子骨骼
```

### 交互功能

#### 搜索功能
```
用户在搜索框输入 "abdomen" 时：
- 自动过滤显示包含 "abdomen" 的骨骼
- 树形结构自动展开相关节点
- 高亮匹配的骨骼名称
```

#### 仅显示有问题的映射
```
☑ 仅显示有问题的映射

勾选后，仅显示：
✗ 父级不匹配的骨骼
✗ 信息不完整的骨骼
✗ 置信度低的骨骼 (<70%)
✗ 用户标记为"需审查"的骨骼
```

#### 自动展开
```
☑ 自动展开

勾选后：
- 所有骨骼自动展开
- 便于全局查看
- 未勾选时记住用户的展开/折叠状态
```

#### 按钮功能
```
[更新时间戳] - 更新配置的修改时间戳
[导出配置] - 将当前配置保存为 JSON 预设
[重置为预设] - 放弃所有编辑，重新加载预设
```

### 详细说明

#### A. 统计摘要

```
未映射骨骼总数: 23 个
  - 可以通过编辑映射来解决的骨骼数

影响顶点组: 5,680 个
  - 有权重数据的顶点组总数
  - 需要权重修复

权重影响: 12.3%
  - 占总权重的百分比
  - 高于 5% 需要立即处理

受影响网格: 3 个
  - 有权重分配给未映射骨骼的网格数
```

#### B. 按分类显示

**分类优先级**：
```
🔴 高优先 (需要立即处理)
  - 权重 > 2%，且无明显映射建议
  - 扭转骨、特殊骨骼

🟡 中优先 (应该处理)
  - 权重 0.5% ~ 2%
  - 控制骨、辅助骨

🟢 低优先 (可以忽略)
  - 权重 < 0.5%，或无权重
  - D-骨、IK 骨、眼睛骨
```

**排序规则**：
```
按权重百分比降序排列
同权重时按顶点组数降序排列
```

#### C. 骨骼卡片

```
#N | 骨骼名称
    🔹 顶点组: XXX 个  |  权重: X.X%  |  ⭐⭐⭐⭐⭐
    父骨: 父级骨骼名
    建议: [ 建议骨骼1 ] [ 建议骨骼2 ]

    ┌──────────────────────────────┐
    │ 在模型中高亮 [◉] | 编辑 [✎] │
    └──────────────────────────────┘
```

**星级评分** ⭐⭐⭐:
```
根据权重百分比和顶点数：
⭐⭐⭐⭐⭐ - 权重 > 5%，顶点 > 1000，立即处理
⭐⭐⭐⭐  - 权重 2-5%，顶点 500-1000，优先处理
⭐⭐⭐   - 权重 0.5-2%，顶点 100-500
⭐⭐    - 权重 <0.5%，顶点 10-100
⭐     - 权重 <0.1%，顶点 <10
```

#### D. 在模型中高亮

当用户点击 **[◉ 在模型中高亮]** 时：

```
1️⃣ 在 3D 视口中：
   ├─ 选中该骨骼对应的顶点组中的所有顶点
   ├─ 高亮颜色: 绿色/黄色/红色（根据权重）
   ├─ 隐藏其他顶点或降低透明度
   └─ 可以 Alt+A 恢复查看所有顶点

2️⃣ 同时在骨骼树中：
   ├─ 自动找到该骨骼的 XPS 对应物
   ├─ 在大纲树中高亮和展开路径
   └─ 显示该骨骼的父级信息

3️⃣ 信息面板显示：
   ├─ 骨骼详细信息（位置、旋转、顶点数）
   ├─ 权重分布直方图
   └─ 建议映射列表
```

#### E. 编辑映射

点击 **[✎ 编辑映射]** 时：

```
快速编辑对话框（不切换到面板 1）：

┌─────────────────────────────────┐
│ 快速编辑: abdomenTwist          │
├─────────────────────────────────┤

```

或者点击 "编辑... (在面板1中)" 以获得完整编辑界面。

#### F. 标记为忽略

点击 **[✗ 忽略]** 时：

```
该骨骼被标记为 "is_ignored=True"
- 卡片变灰色，表示已忽略
- 不再计入统计摘要
- 支持 "撤销忽略" 按钮恢复

确认对话框：
"确定要忽略 'abdomenTwist' 吗？
该骨骼的权重 (2.5%) 将不再修复。"
```

### 交互流程示例

#### 场景 1：修正一个未映射骨骼

```
用户操作：
1️⃣ 在仪表板中看到 "abdomenTwist" 权重 2.5%
2️⃣ 点击 [◉ 在模型中高亮]
   ➜ 3D 视口自动显示该骨骼的顶点（绿色高亮）
   ➜ 可以看到具体有多少顶点被影响
3️⃣ 点击 [✎ 编辑映射]
   ➜ 快速编辑对话框打开，建议映射到 "上半身"
4️⃣ 确认编辑
   ➜ 骨骼立即映射
   ➜ 仪表板自动刷新
   ➜ 该骨骼从列表中移除（已映射）
```

#### 场景 2：批量审查高权重的未映射骨骼

```
用户操作：
1️⃣ 仪表板自动按权重排序，将高权重骨骼排在前面
2️⃣ 用户从上到下逐个审查：
   ➜ 点击 [◉ 在模型中高亮] 查看影响
   ➜ 点击 [✎ 编辑映射] 快速修正
   ➜ 或点击 [✗ 忽略] 标记为无需修复
3️⃣ 完成后点击 [ 📊 生成修复建议 ]
   ➜ 系统为剩余的低权重骨骼生成自动修复建议
```

## 🎨 视觉设计规范

### 颜色方案

```
✓ 正确映射: 绿色 (#4CAF50)
⚠️ 警告/需要检查: 黄色 (#FFC107)
❌ 错误/无映射: 红色 (#F44336)
ℹ️ 信息: 蓝色 (#2196F3)
◯ 中性/背景: 灰色 (#9E9E9E)
```

```python
def draw_bone_tree(parent_bone=None, depth=0):
    """递归绘制骨骼树"""
    children = get_mmd_children(parent_bone)
    for mmd_bone in children:
        # 获取该 MMD 骨骼的 XPS 映射
        xps_mapping = get_xps_mapping(mmd_bone.mmd_name)
        # 绘制骨骼行
        draw_bone_row(mmd_bone, xps_mapping, depth)
        # 递归绘制子骨骼
        draw_bone_tree(mmd_bone.mmd_name, depth+1)
```

**分类逻辑**:
```python
def categorize_unmapped():
    """按分类和权重排序未映射骨骼"""
    high_priority = [b for b in unmapped if b.weight_percentage > 2.0]
    mid_priority = [b for b in unmapped if 0.5 <= b.weight_percentage <= 2.0]
    low_priority = [b for b in unmapped if b.weight_percentage < 0.5]

    # 在每个分类内按权重降序
    return sorted(high_priority) + sorted(mid_priority) + sorted(low_priority)
```

```python
def highlight_vertices(bone_name):
    """在3D视口高亮顶点"""
    mesh = get_mesh_with_vertex_group(bone_name)
    vertices = get_vertices_in_group(mesh, bone_name)

    # 方法1: 选中顶点
    bpy.ops.object.mode_set(mode='EDIT')
    bpy.ops.mesh.select_all(action='DESELECT')
    for vertex in vertices:
        vertex.select = True

    # 方法2: 使用自定义着色器高亮（更好看）
    enable_vertex_color_shader(vertices, color=(0, 1, 0, 0.8))

    bpy.ops.object.mode_set(mode='OBJECT')
```
